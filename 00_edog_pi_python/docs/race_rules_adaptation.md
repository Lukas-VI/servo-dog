# 小型四足规则适配记录

依据 `小型四条腿.pdf` 的图片页人工读取结果，本项目当前按“快递运送赛”定制：

- 场地为黑色底面、白色 500 mm 赛道，机器人从“快递集散中心”出发并回到集散中心结束。
- 单轮任务包括减速带、分岔路、住户投递、上高台、下高台，满分 100 分。
- 两个住户门口以不同颜色条带标识，规则示例为紫色与棕色；快递颜色与住户条带颜色相同时执行投递。
- 比赛总时长 10 分钟，不限轮次；最终成绩等于棕色、紫色各一轮最好成绩之和。同一路分岔重复通过时，只记录一次分岔与快递成绩。
- 手动干预允许但扣分，且干预不得改变机器人与终点之间的有效赛道距离。
- 投递后机器人必须继续稳定行走；动作完成后 3 秒内若失衡、摔倒或需要人工干预，则该障碍失败。

## 系统定制

- Web 首页只保留比赛控制台：左住户任务、右住户任务、直行巡线、结束/急停。
- 左右住户使用独立任务图：`maps/left_resident.json` 与 `maps/right_resident.json`。
- 任务图节点支持拖拽和数值坐标，节点可标注为 `track`、`fork`、`resident`、`marker`、`color`、`action`、`stop`。
- 自动启动前，Web 会先载入对应任务图并保存到 `config.yaml`，再启动后端视觉闭环。
- 投递动作冷却期间状态机继续输出循迹运动，避免只执行投递动作而丢失白色赛道。

## 仍需现场确认

- 紫色住户与棕色住户在真实场地的左右方向可能由赛场摆放决定，赛前应在 Web 首页用“左住户任务/右住户任务”各短跑一次确认。
- 倾斜方向当前保持旧系统映射：紫色触发 `lean_right`，棕色触发 `lean_left`。若球仓安装方向与现场相反，只需在任务图节点的 `action` 字段互换。
- 规则文档中的颜色 RGB 仅为参考，现场应使用 Web 阈值页在当前光照下采样更新 HSV。
